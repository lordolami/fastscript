# FastScript Agent Spec Stack

This stack defines the FastScript coding agent as a product, platform, and long-range model strategy.

Use these files together:

1. `spec/FASTSCRIPT_AGENT_POSITIONING.md`
2. `spec/FASTSCRIPT_AGENT_CAPABILITY_MAP.md`
3. `spec/FASTSCRIPT_AGENT_ARCHITECTURE.md`
4. `spec/FASTSCRIPT_AGENT_DATA_EVAL_AND_TRAINING.md`
5. `spec/FASTSCRIPT_AGENT_EXECUTION_PLAN.md`

This stack answers:

1. why the FastScript agent is different from other coding models
2. what the agent must be able to do now and in the future
3. what architecture must exist for it to work
4. what data, evals, and training loops are required
5. what Codex must implement from A to Z

This is the canonical starting point for all FastScript AI/agent work.

For the compatibility-first product direction and trust contract, also read:

1. `spec/FASTSCRIPT_COMPATIBILITY_FIRST_RUNTIME_SPEC.md`
