# FastScript Agent Execution Plan

## Goal

Provide the exact A-to-Z execution path Codex should follow to build the first serious FastScript coding agent stack.

## Phase 1: Foundation Lock

### A. Ratify the FastScript agent spec stack

Codex must:

1. keep `spec/FASTSCRIPT_AGENT_*.md` as canonical agent planning docs
2. update them when the scope changes
3. avoid building agent features without spec alignment

### B. Confirm current language floor assumptions

Codex must assume the current FastScript `v2.0` floor includes:

1. stable syntax
2. stable semantics
3. stable compiler
4. diagnostics
5. validation
6. exports
7. same-host full-stack primitives
8. predictable structure

## Phase 2: Agent Runtime MVP

### C. Build the first FastScript-native task loop

Codex must implement:

1. prompt -> FastScript generation loop
2. file writer
3. build runner
4. validate runner
5. diagnostics reader
6. repair loop

### D. Build product modes

Codex must define and implement:

1. builder mode
2. developer mode
3. migration mode

## Phase 3: Evaluation Backbone

### E. Create eval harnesses

Codex must implement:

1. prompt-to-page evals
2. prompt-to-app evals
3. repair evals
4. migration evals
5. export evals
6. responsive UI evals

### F. Create score reporting

Codex must emit:

1. pass/fail
2. build result
3. validate result
4. deploy result where relevant
5. prompt-alignment notes

## Phase 4: Data Pipeline

### G. Define dataset schemas

Codex must define machine-readable schemas for:

1. prompt -> FastScript
2. broken -> fixed FastScript
3. JS/TS -> FastScript
4. FastScript -> export target
5. repo + task -> patch

### H. Build data generation/collection tools

Codex must create tools to:

1. collect validated examples
2. filter failed examples
3. score and tag examples
4. prepare corpora for fine-tuning/adaptation

## Phase 5: Export And Product Proof

### I. Strengthen export lanes

Codex must improve:

1. FastScript -> JS
2. FastScript -> TS
3. deploy target outputs
4. future target adapter readiness

### J. Build product proofs

Codex must prove:

1. website generation
2. dashboard generation
3. ecommerce generation
4. full-stack generation
5. repo editing and repair

## Phase 6: FastScript AI Productization

### K. Builder product surface

Codex must help build:

1. prompt-to-app UX
2. preview deploy UX
3. revision-by-prompt UX
4. export UX

### L. Developer product surface

Codex must help build:

1. repo task UX
2. validation loop UX
3. repair/review UX
4. deployment UX

## Phase 7: Specialization

### M. Train/adapt on FastScript corpora

Codex must support:

1. dataset preparation
2. eval preparation
3. benchmark preparation
4. paired export/migration corpora

### N. Optimize for speed and lower-cost inference

Codex must preserve:

1. FastScript-first reasoning
2. short prompts where possible
3. strong validation loops
4. reusable structured app patterns

## Phase 8: Future Targets

### O. Mobile target planning

Codex must later support:

1. FastScript -> mobile target design
2. mobile-specific generation datasets
3. mobile validation and preview loops

### P. Desktop target planning

Codex must later support:

1. FastScript -> desktop target design
2. desktop-specific generation datasets
3. desktop validation and preview loops

## Exit Criteria For First Serious Agent Release

The first serious FastScript agent release is complete when:

1. it can generate FastScript apps from prompts
2. it can repair its own build/validate failures
3. it can edit existing FastScript repos
4. it can migrate meaningful JS/TS slices into FastScript
5. it can export working outputs to the supported lanes
6. it has real evals proving those claims

## Codex Rule

Codex should treat this execution plan as the operational order of work for the FastScript agent unless a later ratified spec overrides it.
