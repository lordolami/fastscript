# FastScript Agent Capability Map

## Scope

This file defines everything the FastScript coding agent must be able to do from the first serious product version through the long-term platform vision.

## A. FastScript-Native Generation

The agent must be able to:

1. create pages in `.fs`
2. create layouts in `.fs`
3. create API routes in `.fs`
4. create middleware in `.fs`
5. create auth/session flows in `.fs`
6. create jobs/workers in `.fs`
7. create DB migrations and seeds in `.fs`
8. create upload/storage handlers in `.fs`
9. create same-host dashboards in `.fs`
10. create reusable UI primitives and product flows in `.fs`

## B. Prompt To Product

The agent must be able to generate:

1. landing pages
2. docs sites
3. blogs
4. admin dashboards
5. SaaS products
6. marketplaces
7. ecommerce stores
8. internal tools
9. content systems
10. multi-role full-stack apps

## C. Design Generation

The agent must be able to:

1. generate a visual system from a prompt
2. generate a responsive layout system
3. generate a typography and color direction
4. create multiple design variants from the same brief
5. redesign existing screens without breaking structure
6. keep desktop and mobile layouts aligned
7. generate token-backed FastScript-friendly styling
8. preserve product intent while changing visual language

## D. Repo Understanding

The agent must be able to:

1. read a FastScript repo
2. map routes and layouts
3. map APIs and middleware
4. map auth and data flows
5. map deploy targets
6. explain structure clearly
7. identify risk areas
8. understand current and target architecture

## E. Code Editing And Refactoring

The agent must be able to:

1. patch existing code
2. refactor routes
3. refactor APIs
4. refactor styling
5. simplify logic
6. split large files
7. merge duplicate logic
8. rename safely
9. preserve behavior while changing structure
10. upgrade patterns to newer FastScript standards

## F. Repair Loop

The agent must be able to:

1. read build failures
2. read validation failures
3. read runtime errors
4. read deploy failures
5. patch broken files
6. rerun checks
7. keep iterating until green
8. explain what was wrong and what was fixed

## G. Review Mode

The agent must be able to:

1. review generated code
2. review FastScript apps
3. find bugs
4. find regressions
5. find missing tests
6. find missing validations
7. rank issues by severity
8. suggest or apply fixes

## H. Migration

The agent must be able to:

1. migrate JS to FS
2. migrate TS to FS
3. migrate route by route
4. migrate APIs
5. preserve UI
6. preserve project structure
7. preserve behavior
8. preserve package compatibility
9. keep mixed-mode migration viable
10. reduce authored JS over time

## I. Export

The agent must be able to:

1. export FS to JS
2. export FS to TS
3. export FS to HTML/CSS/JS style outputs where defined
4. export FS to target scaffolds
5. export FS to Node deploy artifacts
6. export FS to Vercel deploy artifacts
7. export FS to Cloudflare deploy artifacts
8. later export FS to mobile targets
9. later export FS to desktop targets

## J. Multi-Language Reach

The long-term agent must be able to:

1. reason in FastScript first
2. emit FastScript directly
3. emit JS/TS outputs
4. emit framework-style outputs where supported
5. later support broader languages through paired mapping and training

## K. Full-Stack Product Work

The agent must be able to:

1. generate same-host auth
2. generate same-host APIs
3. generate same-host admin tools
4. generate DB flows
5. generate storage flows
6. generate jobs and background workers
7. generate observability surfaces
8. generate deploy configs and target adapters

## L. Builder Mode

The builder-mode agent must be able to:

1. take a prompt
2. generate a product
3. validate it
4. deploy a preview
5. revise by prompt
6. change design by prompt
7. add product features by prompt
8. regenerate dashboards and admin areas by prompt

## M. Developer Mode

The developer-mode agent must be able to:

1. act in an existing repo
2. modify multiple files safely
3. run local commands
4. inspect outputs
5. apply fixes
6. review diffs
7. continue until task completion

## N. Ecommerce Mode

The ecommerce agent must be able to:

1. create storefronts
2. create product grids and detail pages
3. create carts and checkout flows
4. create merchant dashboards
5. create order and inventory workflows
6. create coupon/promo systems
7. create support/admin surfaces
8. redesign storefronts from prompts

## O. Documentation And Ops

The agent must be able to:

1. write docs
2. write release notes
3. write migration guides
4. write deployment docs
5. maintain docs search
6. inspect logs
7. inspect incidents
8. patch production issues

## P. Long-Term General Coding Agent

The long-term system must be able to:

1. use FastScript as the internal reasoning substrate
2. build software in FastScript directly
3. export to target ecosystems
4. migrate non-FastScript projects into FastScript
5. act as a world-class general coding agent while preserving FastScript speed and regularity
